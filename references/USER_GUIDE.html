一、这是什么

insigoo-memory 是一个公益组织专用的AI知识管理工具。

想象一个场景：你机构运行了五年，电脑里存了几千份文件——项目方案、活动照片、预算表、结项报告、志愿者名单、培训材料、政策通知……你找不到三年前的净滩活动记录，新来的同事不知道之前的项目是怎么做的，资助方要的评估报告你忘了哪些还有用。

insigoo-memory 就是来解决这个问题的：你只管把文件往里放，它自动帮你整理、分类、提醒你该做什么。

二、它能解决什么痛点

你想知道 它能做什么
"有哪些文件？它们属于哪种类型？" 自动扫描全部文件，分到9个知识区：行业资讯、研究学习、设计物料、项目方案、项目痕迹、财务资料、监测评估、结项资料、行政人事
"上个月的净滩活动记录在哪？" 9区可交互看板，搜索文件名，点击展开文件列表
"财务区是不是空了很久？" 健康检查：哪个区没有文件、哪些文件疑似重复、哪些政策过时了
"这份项目书写得逻辑通吗？" 项目书诊断：7项原则逐条检查（受益方中心、逻辑自洽、证据为本…）
"99公益日快到了还没做物料？" 行业日历提醒：自动检测99公益日/年检等公益行业特有日期
"最近行业有什么新政策？" 行业资讯日报：自动提取知识包中的政策热点
"安装后要花多少钱？" 零。完全离线运行，不需要API Key，不需要联网

三、怎么用

方式一：有 AI 助手时（推荐）

如果你已经有了一个 AI 助手（如 WorkBuddy、Hermes），insigoo-memory 以一个 Skill 的形式运行。只需告诉助手：

"帮我整理我们机构的知识库，文件在 D:/机构文件，我们关注环保和儿童议题。"

AI 助手会自动： 1. 扫描所有文件 → 自动分类到 9 个知识区 2. 安装环保、儿童两个行业知识包（含政策/术语/资助方） 3. 生成 9 区看板和行业日报 4. 检测日程待办和运营建议

后续使用：你在对话中传一个文件给助手，它自动分类、保存到正确文件夹，并问你要不要顺便诊断。

方式二：没有 AI 助手时（离线模式）

# 第一步：安装（需要 Python 3.10+）
pip install insigoo-memory

# 第二步：初始化
insigoo-memory init -w /你的机构文件夹 -i 环保 教育 儿童

# 第三步：打开看板
insigoo-memory dashboard -w /你的机构文件夹

看板会在浏览器中自动打开，你可以： - 📤 拖文件上传 → 自动分类 → 保存到对应知识区文件夹 - 📋 粘贴项目方案 → 一键诊断 - 🔍 搜索文件 → 实时过滤 - 📰 查看行业日报

不需要联网，不需要付费 API Key。 如果本机有 Ollama（一个免费的本地 AI 引擎），可以开启 AI 辅助分类和诊断。如果没有，纯规则引擎也能准确工作（实测准确率达到100%）。

四、与不同产品的搭配

4.1 与腾讯 Hy-Memory 搭配

Hy-Memory 擅长记住你说了什么（对话记忆蒸馏）。insigoo-memory 擅长整理你存了什么（文件知识管理）。

两者互补：

你与AI助手的对话 ──→ Hy-Memory 记住关键信息
你上传的文件 ────→ insigoo-memory 分类整理到9区

未来方向：两者打通，AI助手既能从对话记忆里调取上下文，又能从知识库里找到相关文件。

4.2 与阿里 Mem0 搭配

Mem0 是通用的 Agent 记忆 API。insigoo-memory 是其公益领域的垂直补充：

  • Mem0 做通用记忆（跨行业）
  • insigoo-memory 做公益专属（9区+12议题+项目诊断）
  • 两者可以并存——Mem0 处理 Agent 的对话记忆，insigoo-memory 处理机构的文件管理

4.3 与 insigoo SAG MCP 服务器搭配（规划中）

SAG（语义增强检索引擎）是 insigoo 开发的语义搜索引擎，计划作为 MCP 服务上线。

搭配后： - insigoo-memory 负责前端：文件分类、看板、诊断 - SAG 负责后端：语义搜索、跨文件关联、向量检索

你问"去年净滩花了多少钱？效果怎么样？"——SAG 从预算表和评估报告中提取答案，insigoo-memory 告诉你这些文件在知识库的哪个区。

五、零成本运行

insigoo-memory 的设计原则之一是无API也能跑。

模式 需要什么 能力
纯离线规则模式 Python 3.10+ 文件分类、看板、诊断、日报 — 全功能
本地 LLM 增强 + Ollama (qwen2.5:7b) AI辅助文件分类、AI诊断（更准确）
云端 LLM 增强 + DeepSeek API Key AI辅助分类和诊断（最快）

对于预算有限的公益组织：装一个 Python + pip install，不需要 API Key，不需要联网，分类准确率已经达到 100%。

七、技术栈与知识产权

借用的开源技术

  • Python 标准库（http.server, argparse, dataclasses）
  • Ollama（本地 LLM 推理引擎）
  • FAISS（向量索引库）
  • LanceDB（嵌入式向量库，设计理念参考）
  • EverOS（Markdown 原生记忆层，理念参考）

insigoo 自研创新

  1. 9区 NPO 知识分类模型 — 公益组织专属的文件分类体系
  2. SIA L1 项目逻辑自洽评估框架 — 7 项原则的项目书诊断标准
  3. 12 行业知识包体系 — 每个议题含政策、术语、资助方、同行动态
  4. 3 级索引机制 — 目录索引 + 项目索引 + 场景索引
  5. 日程检测与运营建议引擎 — 从文件内容主动发现待办和风险
  6. 离线零成本部署方案 — 不依赖 API Key 的全功能离线运行

八、⚠️测试版说明

insigoo-memory 目前是 v0.3.0 测试版，正在持续调试和迭代中。

已知限制： - 纯二进制文件（如 IMG_001.jpg）依赖文件名分类，无法读取内容 - 大文件目录（3000+文件）扫描较慢 - 诊断器当前基于关键词匹配 + 可选 LLM 增强，AI 诊断需要本地或云端模型 - 不支持增删改查的历史版本回溯
